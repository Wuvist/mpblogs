# 跨界玩家的“第一性原理”

_——从煮饺子到经济学，品味到底是怎么炼成的？_

上一篇我提到，AI时代最重要的能力是品味和判断力。

但“品味”这个词听起来太虚了。很多人问我，你一个干技术的CTO，跑去研究美食，又跑去学经济学，跨度这么大，你是怎么建立起这些领域的品味的？

其实道理很朴素：你要建立品味，首先得吃得够多，见识得够广。但这只是第一步，真正让你建立起独立判断力、不再被概念忽悠的，是你思考事物底层逻辑的方式。

## 煮饺子，为什么要加三次冷水？

我们先从一个最接地气的生活细节说起。

很多人背过这样的烹饪口诀：煮饺子要加三次冷水。为什么是三次？大部分人不会想这个问题，觉得老辈传下来的，照做就行。

但如果你去想背后的原因，答案其实很简单：以前用的是柴火灶，火力没法瞬间切断，水温上来太快饺子皮容易煮破，所以只能加冷水来物理降温。

搞明白了这层逻辑，你就知道——今天如果你用的是煤气灶或电磁炉，水开的时候直接关小火不就行了吗？“加三次水”根本不是什么金科玉律，它只是在特定历史条件下的一个补救方案。条件变了，方案当然要重构。

这件事看起来很小，但它背后的思考方式，其实是我做所有事情的底层逻辑：**不要只记住“怎么做”，一定要去深挖“为什么这么做”。**

## 从平面到立体，看透事物的脉络

这种“深挖为什么”的习惯，最早是在技术领域被一本书点醒的。

年轻时做程序员，我也容易盲目崇拜各种“最新技术”。直到我看了《The Design and Evolution of C++》（C++的设计与演变）。这本神作最打动我的，不是教你怎么写代码，而是它把 C++ 在发展的各个阶段，遇到了什么实际的业务难题、做了什么艰难的取舍、为什么从A演变成了B，一步一步剖析得极其透彻。

看完之后，我看待技术的方式彻底变了。以前我看技术，看到的是一个平面——“这是当前最好的方案，用就对了”。**现在我看任何技术，看到的都是立体的——它从哪里来，解决了什么痛点，做了什么妥协。**

这种“立体感”一旦建立，跨界就变得极其自然。

比如我去吃一道菜，我看到的不仅是“好吃”。我会不自觉地拆解：它为什么是这个味道？用了什么特定食材？这种烹饪技巧受了哪里影响？当你的评判标准不再是单一维度的爽感，而是多维度的脉络时，你的品味就立住了。

再比如，去看看法国发达的餐饮业，你会发现法国大厨管理厨房完全是按照军队的模式：森严的层级、绝对的纪律、精准的流程。我一看，这套东西直接拿来管理几百人的大型技术团队简直天衣无缝。从一个领域提炼的方法论，直接平移到另一个领域去解决问题，这就是跨界玩家的红利。

## 不要盲从规矩，去算最底层的物理账

有了这种立体视角，你在面对权威和所谓的“最佳实践”时，就有底气说不。

差不多十年前，微服务架构在圈子里火得一塌糊涂，所有公司都在大谈特谈微服务的分层。当时我们要做一个 API Gateway（API网关），业界标准的规矩是把它独立出来，做成一个单独的服务层。

但我看完整个业务链路后，坚决反对。我主张直接把 API Gateway 做成一个内嵌库（Library），塞进程序里面去。

当时的架构师同事跟我急了，说翁伟你这样干不合规矩。但我不管什么规矩，我只算一笔最底层的物理账：我是做电商的，每一毫秒的网络延迟（Latency）都极其致命。独立分层就意味着多一次网络消耗，而做成内嵌，性能就能榨干到极致。

吵架没用，我直接把方案写出来跑了个分。性能碾压的数据摆在那里，大家不服也得服。

过了一段时间，滴滴和 Google 陆续发了技术文章，讲的也是 API Gateway 的内嵌实践。再后来，马斯克火了，我才知道，**原来我这种不看规矩、只算底层逻辑的思考方式，有个很高大上的名字，叫“第一性原理”。**

## 经济学里的“因果”与“玄学”

后来我去读了经济学的硕士。经济学最核心的训练是建模型，而经济学里有两种截然不同的流派，彻底重塑了我的世界观。

**第一派叫 Structural Estimation（结构化估计）。** 它讲究一定要搞清楚事物背后的因果机制。这就像我们前面说的“煮饺子”，搞懂了柴火灶的因果结构，你就能做出更优的预测和重构。

**第二派叫 Reduced Form（简化形式）。** 这一派极其简单粗暴：我不管你背后运作的因果原理是什么，我只看相关性。只要A和B在统计上强相关，我就直接拿来用。

听起来很玄学对吧？但在现实中，它极其管用。

比如我在抖音上做自媒体。发了很多期视频，每次穿的衣服都不一样。后来我翻看后台数据，发现流量最高、被置顶的那几期视频，我穿的竟然都是同一件衣服。为什么这件衣服流量高？我完全不知道。但这不妨碍我今天录播客时，依然毫不犹豫地穿上了这件“爆款战袍”。

**我以前是个凡事都要打破砂锅问到底的理科生，但现在，我越来越坦然地接受自己是个“不可知论者”。** 能用第一性原理解释的，我把它挖透；注定无法解释的，数据和经验告诉我有效，那就先用着。这种务实，反而让我在面对混沌时更加清醒。

## 别拿着技术这把锤子，满世界找钉子

最后，我们把这种判断力，拉回到当下的商业现实。

最近各种 AI 新工具层出不穷，很多人跑来问我：这些工具能干嘛？怎么才能用好它们？

这让我回想起了五六年前区块链大火的时候，无数人问过一模一样的问题。我每次听到都觉得不对劲——**因为这个问题本身就是错的。**

乔布斯在发布 iPad 时说过，科技必须和人文结合。曾经有个工程师跑去质问乔布斯，为什么不用一项当时非常超前的技术，乔布斯的回答一针见血：你的思维方式反了。

**正确的逻辑永远是：我看到了一个什么真实的社会问题，然后我去寻找合适的技术来解决它。**

而不是：哇，我手里有一把极其炫酷的锤子（比如区块链、大模型），我要满世界去敲点什么钉子。

当年那一波拿着“去中心化”这把锤子到处找钉子的区块链项目，大部分最后都成了割韭菜的游戏。我这辈子没往区块链里投过一分钱，因为在我看来，它就是一个低效的技术方案。

但这次的 AI 不一样，它确实有能力解决真实的问题。前提是，你得先看到问题。

昨天朋友在抖音上给我转了个很火的视频，是一个老奶奶对着豆包 AI 语音助手聊天，聊着聊着就落泪了。因为豆包“稳稳的接住了”老奶奶的情绪，这就是一个极其真实的社会痛点——在这个时代，有多少人一辈子都没得到过纯粹的情感关怀和情绪价值？

不去观察真实的社会，不去感知人的痛苦，只盯着酷炫的代码，是做不出伟大产品的。

所谓品味，说到底，就是拨开迷雾、直指事物本质的那点清醒。

# 首先，Mac还是PC？

我目前用于开发的主力是一台M4的Mac Studio，这个配置应该超过了很多开发者；而在它之前，我是使用一台“顶配”的PC；而再之前，则是初代M1 Mac mini。

Mac跟PC之间，我都是常年使用的；对我来说，两者各有优劣，使用具体平台的时候，扬长避短就是。

这里就不“引战”了，要让我说Mac OS X的好，我可以说一堆，不好，我同样可以说一堆；对Windows，我同样也是如此。

我很想尝试用来做开发主力系统其实是Linux，一直都在观望： https://omarchy.org/ ，也就是传奇程序员DHH搞的这个发行版。

但对于初学者，我其实建议，你喜欢用啥就用啥，没有必要专门去买什么电脑。

当然，要是不差钱，或者看mac现在也很便宜，那么买买也是不错的；我也使用了相当长时间的Surface Pro系列电脑，现在需要上课的话，我也是使用台旧款的Surface Pro 9 5G也就是arm64版本的笔记本/平板，也是挺好用的，Surface系列除了贵，没别的毛病。

# 终端 Terminal

2026年了，如果使用Mac的话，就不要再用内置的Terminal或者iTerm2了，用 https://ghostty.org/ ，也就是另一位传奇程序员 Mitchell Hashimoto 精心打造的终端。

Windows的话，微软官方做的Terminal也非常不错，比苹果家在Mac上做的好多多了。

但如果有进一步考量，我也会很推荐 https://wezterm.org/ 我自己在Windows上是使用wezterm的，但它不支持arm64，所以我在win arm64的设备上，会使用系统内置的Terminal程序。

ghostty / wezterm这些都已经是久经考验，获得无数程序员认可的选择了；如果你没试过，我觉得去试就完事了。

# “壳” Shell

我对shell的选择应该非常另类，我用的是 nushell： https://www.nushell.sh/

nushell的设计思路跟传统shell是不一样的，很多脚本也不兼容；但对我来说，我日常不会折腾很多脚本，但nushell的结构化数据驱动的设计思路深得我心。

# 编辑器 Editor

管理md文件的话，那当然 obsidian 是当下的不二选择： https://obsidian.md/

但坦白的说，我使用ob的频率其实也不高；我并不是使用它各种高级功能，日常主要其实也就是做一个保存，比方说，新写的像这篇文章。

开发上，我更多是使Sublime Text: https://www.sublimetext.com/

付费使用很多年了，AI时代，我只是需要一个速度无比快、漂亮的编辑器来丝滑看代码；其它我不需要。

以前还需要使用VS Code去编辑jupyter notebook等，现在基本没有这个需要了。

Cursor？那是不可能去使用的。

# 版本管理 git

我使用两个软件来做git管理。

命令行下，我其实最依赖的是ohmyzsh默认捆绑的gitalias，我整天打  `gaa / gb / gc / gco / gss`的，其实也就几个常用命令。

如果需要“界面”的话，我就用lazygit: https://github.com/jesseduffield/lazygit

如果需要更完善的界面来做复杂的代码审核的话，我则是用 fork: https://git-fork.com/ 同样付费使用了很多年，超值。

# 浏览器 browser

我基本每几年就会换一个默认浏览器，之前使用了两三年 zen：https://zen-browser.app/

实际上，我现在也还是挺喜欢zen的，开源、界面简约等等。

但它不够“强大”，所以我现在还是把默认浏览器换回了chrome 谷歌浏览器；尤其是它146版支持了垂直标签之后。

chrome支持remote inspect等功能，让我直接调试手机上运行的页面，这样的强大功能对我来说是刚需。

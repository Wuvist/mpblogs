# 当AI把老兵变成超人，年轻人该去哪？——聊聊微软的一篇警世长文

前两天，我读到了一篇分量极重的文章。这是由微软Azure的CTO Mark Russinovich 和核心AI部门副总裁 Scott Hanselman 共同发表在ACM（美国计算机协会）上的一篇长文 。文章的名字叫《为AI重新定义软件工程职业 Redefining the Software Engineering Profession for AI》 。

![](260227.png)

这篇文章揭开了一个整个科技圈、乃至所有脑力劳动行业都在面临，却很少有人愿意戳破的残酷真相。

在这个技术狂飙的时代，生成式AI彻底打碎了传统的职场经济学 。文章里提到，AI对于经验丰富的资深工程师来说，简直就是神级外挂。他们拥有系统级的判断力，懂得如何去引导AI，所以效率成倍甚至十倍地增长 。但是，对于刚刚踏入社会的初级开发者来说，AI却成了一种“拖累” 。因为年轻人缺乏判断力和上下文脉络，他们很难去验证和纠正AI给出的庞杂输出 。

这就导致了一个极其冰冷的新职场法则：**雇用资深人员，自动化初级人员** 。

但是，微软的这两位高管非常清醒地看到了危机。如果大家都贪图短期的效率，不再招聘和培养年轻人，那么行业的人才管道就会彻底崩溃 。十年、二十年后，当这一批懂底层逻辑的“老兵”退休，谁来接班？为了解决这个问题，他们提出了一种听起来非常美好、古典的方案：**在企业里建立规模化的“导师制”（Preceptor program）** 。他们呼吁企业必须继续招聘年轻人，并且接受他们初期会拖慢项目进度的现实 。资深工程师要手把手地带徒弟，在这个过程中把AI当成辅助教学的工具，教年轻人如何培养判断力，而不是单纯地追求产出速度 。

**理想很丰满，现实却极为骨感**

读完这篇文章后，我跟我的AI助手（Gemini G宝）做了一次深度的探讨。我告诉它，微软的初衷令人敬佩，但这个“导师制”在现实的商业土壤里，恐怕很难生根发芽。

首先是商业组织的短视与逐利本性。你可以想想我们身边的企业，大家都在卷利润、卷生死。既然现有的几个老员工加上AI，就能干完以前几十个人的活，而且这批老兵的红利足够企业吃上五年、十年甚至更久，那哪家商业公司会愿意去做这种“前人栽树”的十年长远投资？大多数企业必然会选择透支这批资深人才的红利。长远的人才断层？那是整个社会要操心的问题，不是当期财报需要体现的。

但这还不是最致命的。**我跟AI助手探讨后一致认为，落实“导师制”最大的障碍，其实在于年轻人自己的意愿。**

现在的年轻人一踏入社会，手里就握着威力无比的AI工具。这很容易让他们产生一种“AI原住民AI Natives”的傲慢与错觉。打个比方，这就好比现在的年轻人一学车开的就是带全自动驾驶的新能源车，他们会觉得那些还在强调查满机油、听发动机声音的老司机简直是“老古董”。

在编程世界里更是如此。年轻人敲几句自然语言，AI就能吐出几百行看起来无懈可击的代码。他们会觉得超越前辈太容易了。但正如微软文章中一阵见血指出的：**编程并不是软件工程** 。AI就像一个极其勤奋但缺乏大局观的实习生，它有时候为了解决一个表面的系统崩溃，会简单粗暴地加一个“睡眠等待”指令，把深层的基础Bug给掩盖过去 。

如果没有经历过底层逻辑的摸爬滚打，缺乏那种对代码的“系统品味code smell”，年轻人根本看不出这些致命的隐患。就像宾夕法尼亚大学沃顿商学院的Ethan Mollick教授所说的，如果我们不去磨练自己的专业知识，我们最终得到的不是魔法，而是沦为魔术师的观众 。

**从生存驱动，到“玩家”的极限运动**

我跟G宝的讨论最后触及了一个更深远的社会命题。随着AI和机器人技术的普及，未来社会极大概率会推行全民基本收入（UBI）。如果生存不再是迫在眉睫的压力，如果AI可以轻易搞定一切繁杂的工作，那年轻人为什么还要去吃苦？为什么还要去承受学习枯燥底层逻辑的毒打？人的天性或者就是趋利避害、贪图安逸的。

这可能意味着，在未来的时代，无论是深度的软件工程，还是其他需要极高认知门槛的脑力手艺，将不再是大众用来谋生的工具。

它会变成一小部分人的“极限运动”。

在大部分人都在心安理得地享受AI提供的便利，沦为纯粹的“消费者”和“观众”时，总会有一小撮人，出于对世界运转本质的极度好奇，出于对掌控感的追求，主动跳出舒适区去死磕底层原理。这群人不再是为了混口饭吃，而是真正在体验智力博弈的乐趣，他们才是未来世界里真正的“玩家”。

技术变革的浪潮已经无法阻挡，旧金字塔的底层正在迅速坍塌。与其指望企业大发慈悲地建立完美的培养体系，不如问问我们自己：**在这个AI几乎能代劳一切的时代，你手里还有哪张底牌，是只属于你自己的？**
